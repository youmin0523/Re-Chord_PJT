# Re:Chord Mobile (Capacitor)

Wrap the existing Vite-built web app into iOS and Android native shells.
No code changes to the web app — the same React bundle ships on web,
desktop (Tauri), and mobile (Capacitor).

## One-time setup

```bash
cd frontend
npm install @capacitor/core @capacitor/cli @capacitor/ios @capacitor/android

# initialise native projects (creates ios/ and android/ dirs at repo root
# of frontend/)
npx cap init "Re:Chord" "com.rechord.app" --web-dir dist
npx cap add ios
npx cap add android
```

## Build + sync

```bash
# 1. Web build first
npm run build

# 2. Copy dist/ into the native shells
npx cap sync

# 3. Open in native IDEs
npx cap open ios       # → Xcode
npx cap open android   # → Android Studio
```

## API host configuration

The web app calls `VITE_API_BASE` (default `http://127.0.0.1:7860`).
On a mobile device, `127.0.0.1` is the phone itself — not the dev box.
Two options:

1. **Dev**: set `VITE_API_BASE=http://<your-laptop-ip>:7860` before
   building, and make sure your laptop's firewall lets the device in.
2. **Prod**: point at the public backend URL once Phase B is live
   (e.g. `https://api.rechord.app`).

## Native features available via Capacitor

- **Filesystem** — write downloaded MR / score PDF to the user's documents folder
- **Share** — native share sheet (instead of our web ShareDialog)
- **AudioRecorder** — replaces the MediaRecorder in RecordingPanel on platforms where the web API is unreliable
- **StatusBar / SplashScreen / Haptics** — polish

We feature-detect and prefer native APIs when available, falling back
to the web equivalents otherwise (see `frontend/src/lib/platform.js`).

## Permissions

iOS `Info.plist` and Android `AndroidManifest.xml` need:
- Microphone (recording)
- Network (API)
- Storage (downloads)

These are auto-generated by `npx cap add`; review them once before App
Store / Play Store submission.
