## 1. 처리 목적

Re:Chord(이하 "회사")는 다음 목적을 위해 개인정보를 처리합니다.

| 목적 | 처리 항목 |
|------|-----------|
| 서비스 제공 (음원 분석·악보 생성·챗봇 응답) | 업로드 오디오, 처리 결과, 채팅 메시지 |
| 계정 식별 및 인증 (Phase B) | 이메일, 카카오 OAuth 식별자, 닉네임 |
| 서비스 개선 (AI 모델 평가) | 사용 로그, 작업 메타데이터, 피드백 |
| 장애 대응 및 보안 (Sentry 등) | IP 주소, User-Agent, 오류 스택 트레이스 |
| 법령상 의무 이행 | 거래 기록 (유료 도입 시 별도 안내) |

회사는 위 목적 외의 용도로 개인정보를 이용하지 않으며, 이용 목적이 변경될
경우 사전 동의를 구합니다.

## 2. 수집 항목 및 방법

### 2.1 사용자가 직접 입력·업로드하는 정보

- **오디오 파일 / URL**: 사용자가 변환·분석을 요청한 음원 (mp3/wav/m4a 등,
  최대 N MB). YouTube/SoundCloud 등 URL을 통해 가져온 음원 포함.
- **채팅 메시지**: 음악 코파일럿 챗봇에 입력한 질의 텍스트, 음성 녹음
  (faster-whisper 로컬 전사 후 텍스트만 외부 LLM에 송신).
- **계정 정보 (Phase B)**: 이메일, 닉네임, 카카오 OAuth 프로필 (정식 출시
  시 동의 후 수집).

### 2.2 자동 수집 정보

- 접속 IP 주소, 브라우저 / OS / 디바이스 정보
- 서비스 이용 기록 (요청 시각, 요청 경로, 작업 ID)
- 쿠키 / localStorage (게스트 식별, UI 설정, 챗봇 세션 ID)

### 2.3 수집하지 않는 정보

- 주민등록번호, 외국인등록번호
- 신용카드 정보 (결제 도입 시 PG사가 직접 처리, 회사는 거래ID만 보관)
- 정밀 위치 정보

## 3. 보유 및 이용 기간

| 항목 | 보유 기간 | 근거 |
|------|-----------|------|
| 업로드 오디오 (원본) | 작업 완료 후 **30일** | 사용자 다운로드 가능 기간 |
| 분리된 스템·악보·가사 (출력물) | 작업 완료 후 **30일** | 동일 |
| 작업 메타데이터 (Job 레코드) | **1년** | 서비스 개선·통계 |
| 채팅 세션 로그 | **1년** (사용자 삭제 시 즉시 파기) | 컨텍스트 유지·품질 개선 |
| 접속 로그 (IP·UA) | **3개월** | 통신비밀보호법 시행령 §41 |
| 계정 정보 (Phase B) | 탈퇴 시까지, 탈퇴 후 **즉시 파기** (단, 부정이용 방지를 위해 식별자 해시만 30일 유지) | PIPA §21 |

기간 경과 시 회사는 해당 정보를 복구 불가능한 방식으로 **자동 파기**합니다.
파기 절차는 §8 참조.

## 4. 제3자 제공 및 처리 위탁

### 4.1 제3자 제공

회사는 사용자의 사전 동의 없이 개인정보를 제3자에게 제공하지 않습니다.
다만 다음의 경우 예외로 합니다:

- 법령에 따라 수사기관·법원이 적법한 절차로 요청한 경우
- 사용자의 생명·신체에 급박한 위해를 방지하기 위해 필요한 경우

### 4.2 처리 위탁 (출시 시점 확정)

| 수탁자 | 위탁 업무 | 이전 정보 | 위치 |
|--------|-----------|-----------|------|
| OpenAI, L.L.C. | 음악 챗봇 LLM 추론 | 채팅 메시지 텍스트, Job 컨텍스트 메타 (key/BPM/코드/섹션) | 미국 |
| Anthropic / Google (선택적) | 동일 | 동일 | 미국 |
| Cloudflare, Inc. (R2 스토리지) | 분리된 오디오 출력 파일 보관 | 처리 결과물 | 한국·아시아 리전 |
| Supabase, Inc. (Phase B) | 계정 인증·DB 호스팅 | 이메일, 카카오 식별자 | 한국 리전 (예정) |
| Sentry.io | 서버 오류 모니터링 | IP, User-Agent, 스택 트레이스 | 미국 |
| Google Cloud Platform (Cloud Run) | API 서버 호스팅 | 처리 중 메모리상 정보 | 한국 (asia-northeast3) |

**중요**: 채팅 메시지와 Job 컨텍스트 메타는 OpenAI/Anthropic의 모델 학습에
사용되지 않도록 API 설정합니다 (OpenAI API: `X-OpenAI-Data-Opt-Out` 또는
zero-data-retention 계약).

### 4.3 국외 이전 고지

위 수탁자 중 미국 소재 회사로 정보가 이전됩니다. 이전 시점·항목·근거·
수탁자 연락처는 사용자 가입 동의 시 별도 화면으로 명시합니다 (PIPA §28-8).

## 5. 정보주체의 권리

사용자는 언제든지 다음 권리를 행사할 수 있습니다:

| 권리 | 행사 방법 |
|------|-----------|
| 열람 | 마이페이지 → "내 작업 기록" / 챗봇 설정 → "대화 내보내기" |
| 정정·삭제 | 작업 페이지의 휴지통 / 챗봇 사이드바의 삭제 메뉴 |
| 처리 정지 | 이메일(TBD)로 요청 → 7일 내 처리 |
| 동의 철회 | 계정 탈퇴 (Phase B) / 게스트 모드는 브라우저 localStorage 비우기 |
| 권리침해 신고 | 개인정보분쟁조정위원회 (1833-6972), 개인정보침해신고센터 (KISA 118) |

미성년자(만 14세 미만)의 경우 법정대리인의 동의가 필요합니다. 회사는
가입 시 연령을 확인하며, 미성년자의 가입이 확인된 경우 법정대리인 동의
없이 처리한 정보는 지체 없이 파기합니다.

## 6. 안전성 확보 조치

회사는 PIPA §29 및 안전성 확보조치 기준 고시에 따라 다음 조치를 시행합니다:

- **암호화**: 통신 구간 TLS 1.2 이상, 저장 시 AES-256 (오디오는 클라우드
  스토리지의 서버사이드 암호화 사용)
- **접근 통제**: 운영 서버는 SSH 키 + IP 화이트리스트, 데이터베이스는
  IAM 역할 기반 접근, 작업 자료는 작업 소유자만 다운로드 가능
- **로그 보관 및 점검**: 접근 로그 1년 보관, 분기별 검토
- **악성코드 방지**: 컨테이너 이미지 정기 스캔, 의존성 lockfile 고정
- **개인정보 침해 사고 시 신고**: 사고 인지 후 24시간 내 사용자에게
  통지, 72시간 내 KISA에 신고 (정보통신망법 §48-3)

## 7. 쿠키 / 로컬 스토리지

- 게스트 식별자 (브라우저 UUID)
- UI 설정 (다크/라이트 테마, 언어)
- 챗봇 세션 ID 및 히스토리
- 음성 입력 시 임시 녹음 데이터 (브라우저 메모리에서만, 전사 후 즉시 폐기)

사용자는 브라우저 설정에서 쿠키·로컬 스토리지를 차단할 수 있으나, 차단
시 게스트 모드의 챗봇 히스토리와 라이브러리 기능이 동작하지 않습니다.

## 8. 파기 절차

| 단계 | 처리 |
|------|------|
| 파기 사유 발생 | 보유 기간 도과 / 사용자 삭제 요청 / 탈퇴 |
| 파기 시점 | 사유 발생일로부터 **즉시 (자동 배치)** 또는 **5일 이내 (수동)** |
| 파기 방법 (전자) | 영구 삭제 명령 (예: `shred` / 클라우드 stora ge object delete + version pruning) |
| 파기 방법 (출력물) | 해당 없음 (모든 데이터는 전자적) |
| 파기 확인 | 월 1회 파기 로그 감사 |

## 변호사 검토 시 우선 확인사항 (작성자 메모)

1. 채팅 메시지의 LLM 송신을 "처리위탁"으로 볼지 "제3자 제공"으로 볼지
   (KISA 가이드라인의 SaaS API 위탁 기준)
2. OpenAI/Anthropic API에 송신되는 Job 컨텍스트가 "민감정보"에 해당하지
   않는지 — 워십 신앙 정보가 PIPA §23 민감정보(종교) 해당 가능성
   → **민감정보라면 별도 동의 필수**
3. 오디오에 포함된 보컬이 화자의 음성정보로서 PIPA §22-2 (생체정보) 적용
   여부
4. R2/S3 국외 이전 시 사용자에게 가입 화면에서 별도 동의 받는 UI 구현
5. 음성 인식(Whisper) 로컬 처리가 자동수집정보 동의에 포함되는지
6. GDPR 대응이 필요한 경우 §13/§14 정보 제공 의무 추가
